01

背景Background

GRPO/DAPO 以一组 K 个 rollout 为单位更新。一组全对 / 全错 → 组内零方差 → 梯度为 0。「有对有错」的概率 UK(p) = 1 − pK − (1−p)K 在 p = ½ 取最大。 RLVE [arXiv:2511.07317] 用「acc ≥ 0.9 才升一档」决定课程 —— 工作点偏高、信号偏弱、只升不降。

方法升档触发工作点 pUK (K=4)梯度信号课程动态
RLVE-90 baseline acc ≥ 0.9 ~0.7 – 0.9 0.57 弱 只升
Signal-RLVE min −log UK 0.50 0.94 最强 可升可降
FEP-RLVE min G(d) 0.50 + info 0.94 最强 + 探索 可升可降
02

想法Idea

把「选哪个难度训练」重述为 Active Inference 的行动选择：最小化 期望自由能 G(d) —— 同时拿到「偏好风险（让一组有对有错）」与「信息增益（弄清模型能力）」两项。难度自动贴着能力前沿。

怎么做Method

G(d) = λpref·(−log UK(p̂)) 实用 − λinfo·I(s;o|d) 认知 + λcost·C(d) 代价
π(d) ∝ exp(−G(d) / T)
p̂ = 𝔼q[σ(a(s−d))]预测成功率（IRT 似然）
UK一组 K 个 rollout 「有对有错」的概率
I(s;o|d)对能力信念 q(s) 的信息增益
T温度：T→0 贪心，T→∞ 均匀探索
FEP-RLVE Algorithm Ours
Require: 能力信念 qe(s) ; λpref, λinfo, λcost, T, K
1repeat
2采样环境 e
3for d = dmin … dmax do逐档估 EFE
4p̂ ← 𝔼[σ(a(s−d))]预测成功率
5U ← 1 − p̂K − (1−p̂)K有效率
6I ← H[Ber(p̂)] − 𝔼s H[Ber(σ(a(s−d)))]信息增益
7G ← −λpref·log U − λinfo·I + λcost·CEFE
8end for
9d ∼ softmax(−G(·)/T)可升可降
10rollouts ← πθ ; r ← verify
11θ ← GRPO-update(rollouts, r)RL 内核不变
12qe(s) ∝ qe(s)·P(o|s,d)贝叶斯更新
13until 训练结束

Analysis · 关键观察
  • 训练成功率最高的不是泛化最好的。 RLVE-90 训练 p 最高（0.55）但 held-out 几乎不动。
  • 对准 ½ 信号最优带是有用的：Signal-RLVE 提升 +0.11，是 RLVE-90 的 5× 以上。
  • EFE 的认知项需要时间：FEP-RLVE 同样改善（+0.03），其信息增益项在 30 步 / 60% 截断下尚未完全体现。
  • RL 内核与算力开销几乎不变：只增加 O(难度数) 的 CPU 计算。
