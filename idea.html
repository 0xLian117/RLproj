FEP-RLVE:把难度选择
变成 active inference 的行动决策

RLVE 用启发式调难度:准确率 ≥ 0.9 才升一档。我们把「选哪个难度训练」当作 active inference 里的行动选择:为每个环境维护一个对「模型能力」的信念, 选期望自由能(EFE)最小的难度——同时兼顾学习信号(pragmatic)与 能力信息增益(epistemic)。

基线 · RLVE (arXiv:2511.07317) 环境 · RLVE-Gym(400 可验证环境) 模型 · ProRL-1.5B-v2 算力 · 单卡 H200
00

一句话定位

在「可验证环境 + 强化学习」这条路线上(RLVR → RLVE),程序能无限出题、用算法 verifier 自动判分,难度由一个旋钮 d 控制。 本 proposal 复现 RLVE,并重新设计其中的「难度采样器」:把 RLVE 的工程启发式(准确率≥0.9 升难), 升级为 FEP / active inference 的期望自由能(EFE)行动选择。EFE 只决定「下一批题从哪个难度来」,RL 算法仍用 DAPO。

01

前因:为什么固定难度会「学不动」

GRPO/DAPO 的训练单元是一组 rollout:对同一道题采 G 个回答(典型 G=8),每个由 verifier 判对(1)/错(0), 再在组内做相对优势来更新策略。一个关键的、决定成败的性质:

失效机制
一组回答若全对或全错,组内奖励没有差异 → 优势恒为 0 → 这一组贡献的梯度为零,算力被浪费。 DAPO 干脆把这种「零方差组」直接丢弃。

设模型在某难度的答对概率为 p。组内奖励方差是 p(1−p),在 p=0.5 时最大; 「这一组有对有错(可用)」的概率为 U = 1 − pᴳ − (1−p)ᴳ,同样在 0.5 最大。直观代入(G=8):

p=0.95（太易）→ 整组全对 ≈ 0.95⁸ ≈ 66% 的组白采 p=0.50（适中）→ 全对或全错各 ≈ 0.4% → 99% 的组有效 p=0.10（太难）→ 整组全错 ≈ 0.9⁸ ≈ 43% 的组白采

结论:固定难度必然失败。太易 → 很快全对、信号枯竭;太难 → 长期全错、几乎不更新。 只有把成功率维持在 0.5 附近,梯度信号才持续存在。这正是「可验证环境 + 自适应难度」这条路线要解决的核心。

02

原方法:RLVE 怎么做、用什么数据

数据 — RLVE-Gym:手工设计的可验证环境

RLVE 手工设计了 400 个可验证环境(RLVE-Gym)。每个环境是一段程序,包含:

  • 可验证:算法 verifier 直接判对错,无需人工标注答案;
  • 可控难度:一个难度等级 d(d≥0),越大越难;
  • 无限实例:在任意难度下程序化生成无穷多道题。

另留 50 个 held-out 环境测泛化;模型从 ProRL-1.5B-v2(本工作的起点)等强 1.5B 推理模型继续训练;评测:AIME24/25 · OMEGA · OlympiadBench · LiveCodeBench · BBEH。

难度控制器 — 准确率达标才升难(单向 bump)

RLVE 每个环境维护一个难度区间 [ℓ, h],按 d∼Uniform(ℓ,h) 出题;在上界 h 统计准确率 a/b,当样本够多(τ_num)且 a/b ≥ τ_acc=0.9 时升一档:

若 a/b ≥ τ_acc(=0.9): h ← h + 1 （滑窗 d_Δ=4,只升不降）

问题在哪(本 proposal 的出发点)

  • 工作点是 0.9,不是 0.5:它把模型停在准确率 0.9 才升档,而学习信号 U(0.9)≈0.57 ≪ U(0.5)≈0.99(G=8)——大量梯度信号被白白浪费在低信号区。
  • 只升不降、是启发式而非目标:没有被优化的目标函数;模型回退时不会降难,难以分析与推广。
  • 缺乏显式探索:盯单一难度点,没有「多样性/探索」项,也不会主动放弃已饱和的难度。
  • 阈值写死:τ_acc=0.9 固定;奖励一旦变成「通过测例比例」等连续形式,最优工作点会偏移,定阈值无法自适应。
03

如何深入:把难度选择写成 EFE 最小化

把 RLVE 翻译成 active inference 的语言 —— 「选哪个难度训练」是一个行动选择问题:

Active inference难度自适应里的对应物
hidden state s模型在某环境的真实能力
belief q(s)当前对模型能力的估计(带不确定性)
observation o该难度 rollout 做对 / 做错
action / policy π选哪个难度 d
expected free energy G(d)选难度 d 的预期代价(信号 + 信息 + 成本)

每个候选难度 d,算它的期望自由能 EFE,按 softmax 采样(active inference 标准做法 π∝exp(−G/T)):

Ge(d) = − λsignal·UK(p̂) − λinfo·I(s;o|d) + λcost·C(d) π(d|e) ∝ exp( − Ge(d) / T )

两项价值:pragmatic + epistemic

  • pragmatic(学习信号) UK(p)=1−pK−(1−p)K: 一组 K 个 rollout「有对有错、能产生梯度」的概率(= RLVE/DAPO 的 effective prompt ratio),p=0.5 时最大。
  • epistemic(信息增益) I(s;o|d)=H[q(s)]−EoH[q(s|o,d)]: 出这道题能多大程度降低对模型能力的不确定性。太易/太难都无法区分能力,边界难度(p≈0.5)信息增益最大。
两个独立理由都指向 0.5,且 explore→exploit 自动发生
  • pragmatic 的 UK 与 epistemic 的信息增益都在 p≈0.5 峰值——梯度方差视角与信息论视角殊途同归(已在代码中数值验证)。
  • 信念不确定时(早期/能力刚变):epistemic 主导 → 广探以定位能力;信念定准后:pragmatic 主导 → 利用最优带。 无需手调温度退火,explore→exploit 从 EFE 自然涌现。
  • 令 λinfo=0 即退化为「只对准信号带」的 Signal-RLVE 消融;FEP-RLVE 比它好,才证明信息增益(FEP 的核心)真有贡献。

能力信念怎么来(IRT):似然 P(对|s,d)=σ(a(s−d));每次 rollout 做对/做错后对 q(s) 做贝叶斯更新——这就是 FEP 的 belief update。

具体怎么操作:在线 EFE 控制器 + 落地

每个训练步,对每个环境:

1. 对各难度 d 算 G(d) = −λsigUK(p̂) − λinfoI(s;o|d) + λcostC(d) 2. 采样 d ~ softmax(−G/T),驱动 Gym 控制器到该难度并出题 # 行动 3. rollout + verifier 判对/错 # 观测 o 4. 对能力信念 q(s) 做贝叶斯更新 # belief update

FEP 只控制「下一批题从哪个难度来」,不改 RL 算法;RLVE 本身对 RL 算法无要求,仍用 DAPO。

工程落地(已对真实 RLVE 代码实现并离线验证)

  • 难度调度全在 slime/ray/rollout_data_source.py 的 RLVEManager(generate_problem()/update())。
  • FEPRLVEManager(RLVEManager) 只重写这两个方法:Gibbs 采难度 + 信念贝叶斯更新; apply_patch.py 用 env 变量 DIFFICULTY_MODE 切换(默认逐字保持 RLVE 原行为)。
  • 超参经环境变量:FE_DMAX / FE_K / FE_SLOPE / FE_LSIG / FE_LINFO / FE_LCOST / FE_T。
05

实验设计与验证

数据划分

从 RLVE-Gym 取训练环境(ProRL-1.5B-v2 起点),另用 held-out 环境 + 公开 benchmark(AIME/OMEGA/OlympiadBench/LiveCodeBench/BBEH)评训练前/后,衡量泛化。

四个对照臂(同模型、同算力,只换难度采样器)

臂难度策略隔离了什么
Static难度焊死固定难度会失败(饱和/停滞)
RLVE-90acc≥0.9 升难(基线)停在 0.9 高成功率(低信号)
Signal-RLVE只 UK(λinfo=0)对准 0.5 信号带,但无信息增益
FEP-RLVE(本工作)UK+信息增益+EFE完整 active inference

为什么必须有 Signal-RLVE:只有 FEP-RLVE 比它还好,才证明 epistemic 信息增益(FEP 的核心)真有贡献,而不是「只是把 0.9 改成 0.5」。这一对照是「算不算 FEP」的命门。

核心指标

  • effective prompt ratio(机理):UK(p̂) 实测均值。预期 Static 塌、RLVE-90≈0.57、Signal/FEP≈0.99。
  • 能力信念轨迹(仅 FEP):q(s) 的均值/标准差;σ 收缩 = 主动定位能力。
  • 泛化:held-out 环境 + benchmark 训练前后 Δ;主张 FEP ≥ Signal > RLVE-90 > Static。
  • 消融:扫 λinfo、T。
已离线验证(控制器核心)
UK 与信息增益都在 p≈0.5 峰值;能力信念跟踪上升能力、标准差从 1.2 收缩到 0.06(explore→exploit)。 RLVE 上跑完后真实数据回填此处。

诚实的边界 / 失败情形

当模型能力超过 d_max,所有难度饱和、信号枯竭——任何控制器都无能为力,反过来说明需要扩充环境(env scaling),呼应 RLVE 的核心论点,也是本工作的自然延伸。

06

贡献小结

  • 从 rule-based 到 belief-based 课程:把 RLVE 的「acc>0.9 升难」换成 π(d)∝exp(−G(d)/T)。
  • 把「适中难度」定义为期望自由能 G=−λsigUK−λinfoI+λcostC,UK 直接对齐 DAPO/RLVE 的有效训练 prompt。
  • 维护 latent competence 信念:难度控制从「滑窗准确率」升级为「主动推断模型能力」(FEP),并以 Signal-RLVE 消融隔离信息增益贡献。
