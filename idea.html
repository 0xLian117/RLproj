给可验证环境的 RL 一个「自由能」难度控制器

在 SCALER 之上的创新点 · 把「固定 0.5 成功率」推广成一个有理论的目标

1. 背景:RL 为什么会「学不动」

用 RL 训模型做推理,每道题会采 G 个回答(比如 G=8),用程序自动判每个对(1)/错(0), 再用 GRPO 在组内做对比来更新。关键事实:

一组回答如果「全对」或「全错」,组内没有差异 → 梯度为 0 → 这组白采。
只有「有对有错」的组才提供学习信号。

设这道题答对概率为 p,组内奖励方差是 p(1−p),在 p=0.5 时最大; 「一组有对有错」的概率是 U = 1 − pᴳ − (1−p)ᴳ, 同样在 0.5 最大。难度刚好让模型对一半,信号最强。

2. SCALER 做了什么

SCALER(我们复现的框架)把真实代码题合成成可验证环境:程序无限出题、用单元测试自动判分、 用一个旋钮 d 控制难度。它的难度控制器是一个定点调节:

测到成功率偏离 0.5 → 把难度 d 往回调,使 success → 固定的 0.5

即:只盯住「成功率 = 0.5」这一个目标点。简单有效,但它没有一个被优化的显式目标, 也没说为什么是 0.5、以及难度该如何在多个档位间分配。

3. 我们的想法:不盯一个点,而是优化一个分布(自由能 + Gibbs)

把每个难度档的「学习价值」记为 U(d)=1−p(d)ᴳ−(1−p(d))ᴳ。 与其塌到单个最优点,我们在所有难度档上选一个采样分布 q(d),最小化自由能:

F[q] = − Eq[U] − T · H[q] （H 是熵,T 是温度）

能量项 −E[U]:把概率投向学习价值高的难度。 熵项 T·H:保持多样性、不要孤注一掷。两者最优折中的解,正是物理里的 Gibbs / 玻尔兹曼分布:

q*(d) ∝ exp( U(d) / T )
Gibbs 是什么

一句话:「价值越高的选项,概率指数级越大;温度 T 控制这个偏好有多极端」。

  • T → 0(低温):全部概率压到价值最高的那个难度 = argmax U(d) ≈ 成功率 0.5 那一档 —— 这就退化成 SCALER 的定点。所以 SCALER 是我们框架的一个特例。
  • T → ∞(高温):各难度均匀采样 = 最大探索 / 难度多样。
  • 中间 T:既聚焦最有信息的难度,又顺带探索邻近难度;还会自动放弃「已饱和(p→1)」和「学不动(p→0)」的难度(它们 U 很低)。

额外好处:同一个目标若写成 U(环境,难度),它的「环境维度边缘分布」就是环境选择权重 —— 难度控制和环境取舍由一个目标统一导出。

5. 实验:5 个臂的对照(同模型 / 同算力)

臂难度策略预期现象
static-易难度焊死(低)很快全对 → 信号塌掉(有效样本率→0)
static-中 / 难难度焊死(中/高)偏难 → 大量全错,学得慢
SCALER-adaptive定点 0.5维持中等难度,信号较高
free-energy(我们)q(d)∝exp(U/T)有效样本率最高、泛化最好(待验证)

核心度量 = 有效样本率(每步「有对有错」的组占比);外加训练前后在 MATH-500 / AIME / AMC / MMLU-Pro / BBEH / GPQA 上的泛化。

合成数据初步验证(同格式):static-易 0.22 / static-难 0.49 / adaptive 0.99 —— 自适应把信号维持得最高,正是我们要证明的机理。

6. 一句话总结

SCALER 把难度盯在「成功率 0.5」;我们把它推广成最小化自由能 F=−E[U]−T·H, 最优解是 Gibbs 分布 q(d)∝exp(U(d)/T)。一个温度旋钮 T 统一了「利用 vs 探索」和「难度 vs 环境选择」, T→0 就还原成 SCALER,中间温度则维持更多学习信号。
