FEP-RLVE:用自由能原理选择训练难度
把强化学习的「难度自适应」从一条写死的规则,变成一个可推导的决策目标

一句话总结
我们在复现 RLVE 的基础上,把它「准确率超过 0.9 就把题目调难一档」这条人工规则, 替换成一套基于自由能原理(Free Energy Principle)的难度选择器: 它一边估计「模型现在到底有多强」,一边挑出最能产生学习信号、又最能减少这个估计不确定性的难度去训练。

1背景:我们在解决什么问题

用强化学习(RL)训练大模型做推理,核心难点不是「缺题」,而是缺能持续产生学习信号的「适中难度」题目。 原因在于现在主流的 RL 算法(GRPO / DAPO)是按「一组」来更新的:对同一道题采 K 个回答(比如 K=8), 每个判对(1)或错(0),再用组内的相对好坏来更新模型。

关键失效机制
如果这一组 K 个回答全对或全错,组内没有差异 → 算出来的优势全为 0 → 这一组完全不产生梯度,算力白费。(DAPO 干脆把这种「零方差组」直接丢掉。)

设模型在某难度的答对概率是 p。一组「有对有错、能用」的概率是

UK(p) = 1 − pK − (1−p)K　在 p = 0.5 时最大

直观代入(K=8):太简单 p=0.95 → 整组全对约 66%,大半白采;太难 p=0.1 → 整组全错约 43%; 只有 p=0.5 时约 99% 的组都有效。结论:难度必须维持在「模型刚好对一半」的地方,信号才最强。

2以往的方法(RLVE)及其问题

RLVE 怎么做

RLVE(我们复现的基线工作)手工设计了 400 个可验证环境(RLVE-Gym):每个环境是一段程序, 能无限出题、用算法自动判对错、并用一个旋钮 d 控制难度。它让难度自适应: 每个环境维护一个难度上界 h,当模型在 h 处的准确率 ≥ 0.9 时,就把 h 调高一档(只升不降)。

问题在哪

  • 工作点定在 0.9,而不是信号最优的 0.5:模型停在准确率 0.9 才升难,而 UK(0.9)≈0.57 ≪ UK(0.5)≈0.99,大量梯度信号被浪费在低信号区。
  • 是一条写死的规则,不是可优化的目标:阈值 0.9 是拍脑袋定的;只升不降;无法分析、无法推广。
  • 没有「探索」与「不确定性」的概念:它只盯一个难度点,不会主动去试探「模型到底强在哪、弱在哪」。

3自由能原理是什么 · 我们怎么参照

自由能原理(Free Energy Principle, FEP) / 主动推断(Active Inference)是认知科学家 Friston 提出的一套理论, 用来解释「智能体如何行动」。它的核心主张是:智能体选择行动,是为了最小化「期望自由能(Expected Free Energy, EFE)」。 而 EFE 同时包含两种价值:

  • 实用价值(pragmatic value):这个行动有没有帮我达成目标 / 拿到回报;
  • 认知价值(epistemic value):这个行动能不能帮我减少对世界的不确定性(获取信息)。

一句话:好的行动 = 既奔着目标去,又顺便把「我对环境的认识」搞得更清楚。这正是人主动选择「能学到东西」的活动的原理。

怎么参照到我们的问题
我们发现:「选哪个难度训练」本质上就是一个「行动选择」问题,而不是简单的打分。于是直接套用 active inference 的语言:
自由能原理里的概念我们难度自适应里的对应物
隐藏状态 s模型在某环境上的真实能力
信念 q(s)我们当前对模型能力的估计(带不确定性)
观测 o某难度的题做对 / 做错
行动 / 策略 π选择哪个难度 d 出题
期望自由能 G(d)选难度 d 的预期「代价」(信号 + 信息 + 成本)

4我们的方法:怎么把自由能引入 RLVE

不再用「acc≥0.9 升一档」,而是对每个候选难度 d 计算它的期望自由能 G(d),再按 active inference 的标准方式(softmax)采样:

Ge(d) = − λsignal·UK(p̂) − λinfo·I(s;o|d) + λcost·C(d)
π(d|e) ∝ exp( − Ge(d) / T )

三项的含义:

  • 实用价值 UK(p̂) = 前面那个「一组有对有错」的概率(就是 RLVE/DAPO 里说的有效 prompt 比例),在 p=0.5 最大 → 鼓励挑「信号最强」的难度。
  • 认知价值 I(s;o|d) = 出这道题能减少多少对模型能力的不确定性(信息增益)。太易/太难都无法区分能力,只有边界难度(p≈0.5)信息增益最大。
  • 成本 C(d)(可选):生成/验证开销,防止总挑特别贵的题。
两个独立理由都指向 0.5,且「先探索后利用」自动发生
  • 实用价值(梯度方差视角)和认知价值(信息论视角)各自独立地都在 p≈0.5 处最大 —— 互相印证。
  • 当我们对模型能力还不确定时(训练早期、或能力刚变化),认知项占主导 → 广泛试探不同难度去「摸清模型能力」;一旦能力估计变准,实用项占主导 → 集中在最优难度训练。这种 explore→exploit 是从自由能目标里自然涌现的,不用手动调。
  • 温度 T:T→0 退化成一个固定的 0.5 定点控制器(已经比 RLVE 的 0.9 更优);T→∞ 则均匀探索所有难度。

能力信念怎么估计、怎么更新

用类似心理测量学 IRT 的模型:做对的概率 P(对 | 能力 s, 难度 d) = σ(a·(s − d))。 每次 rollout 看到做对/做错,就对能力后验 q(s) 做一次贝叶斯更新 —— 这正是自由能原理里的「信念更新」。

5创新点小结

  • 从「按规则」到「按推断」的课程:把 RLVE 的 acc≥0.9 升难,换成 π(d)∝exp(−G(d)/T) 的概率采样。
  • 把「适中难度」定义成期望自由能:用 UK 直接对齐 DAPO/RLVE 的有效训练 prompt,理论上可推导、可消融。
  • 维护模型能力的隐变量信念:难度控制从「滑窗准确率」升级为「主动推断模型能力」——这是 FEP 的核心,也是与 RLVE 最本质的区别。

6用什么模型、怎么做实验

模型:从 ProRL-1.5B-v2(NVIDIA Nemotron-Research-Reasoning-Qwen-1.5B 的 v2,RLVE 论文用的强 1.5B 推理模型起点)继续训练。 框架:官方 RLVE(SLIME 后端),单卡 H200。所有臂同模型、同算力、同 RL 算法(DAPO),只换难度采样器。 我们的控制器是一个 drop-in 替换,用环境变量切换,不设时逐字保持 RLVE 原行为(保证对照公平)。

对照臂难度策略用来检验
Static难度冻结不变固定难度会饱和 / 停滞
RLVE-90acc≥0.9 升难(基线)停在 0.9 低信号区
Signal-RLVE只用 UK(λinfo=0)对准 0.5,但无信息增益
FEP-RLVE(本工作)UK + 信息增益 + EFE完整的自由能控制

为什么必须有 Signal-RLVE:只有 FEP-RLVE 比它还好,才能证明「信息增益(FEP 的核心)」真的有贡献,而不是「只是把 0.9 改成 0.5」这么简单。这是「算不算 FEP」的关键消融。

看什么指标

  • 有效样本率 UK(p̂):预期 Static 塌、RLVE-90≈0.57、Signal/FEP≈0.99。(核心机理证据)
  • 能力信念轨迹:FEP 臂的能力估计均值/标准差,σ 收缩说明在主动定位能力。
  • 泛化:训练前后在 held-out 环境 + 公开 benchmark 上的提升,主张 FEP ≥ Signal > RLVE-90 > Static。
目前进展(已验证)
控制器已离线验证:UK 与信息增益都在 p≈0.5 峰值;能力信念能跟踪上升的能力,标准差从 1.2 收缩到 0.06(即先探索后利用)。 RLVE 复现已在 ProRL-1.5B-v2 上正常训练(约 40 秒/步,平均组奖励约 0.41)。〈四臂完整曲线与泛化数据,待训练日志回填。〉

7结论与边界

把难度选择写成期望自由能最小化,就用一个有理论依据、单一温度旋钮的控制器,替代了 RLVE 拍脑袋的 0.9 阈值: 它对准信号最优的工作点、并主动推断模型能力,在同等算力下维持更多学习信号。

诚实的边界:当模型能力超过环境的最高难度 dmax,所有难度都饱和、信号枯竭,任何控制器都无能为力 —— 这反过来说明需要扩充环境(environment scaling),正好呼应 RLVE 的核心论点,也是本工作的自然延伸。
