用自由能把「难度自适应」
从一个定点变成一个目标

SCALER 把训练难度调到「成功率 ≈ 0.5」这一个固定点。我们提出:把难度选择写成一个自由能最小化问题, 其最优解是一个由温度控制的 Gibbs 分布——SCALER 只是它在零温下的特例。

00

一句话定位

在「可验证环境 + 强化学习」这条路线上(RLVR → RLVE → SCALER),程序能无限出题、用算法/单元测试自动判分,难度由一个旋钮 d 控制。 本 proposal 不改这套环境与训练框架,而是重新设计其中的「难度控制器」:从一个工程启发式,升级为一个有显式目标、可推导、可退化回基线的控制律。

01

前因:为什么固定难度会「学不动」

GRPO/DAPO 的训练单元是一组 rollout:对同一道题采 G 个回答(典型 G=8),每个由 verifier 判对(1)/错(0), 再在组内做相对优势来更新策略。一个关键的、决定成败的性质:

失效机制
一组回答若全对或全错,组内奖励没有差异 → 优势恒为 0 → 这一组贡献的梯度为零,算力被浪费。 DAPO 干脆把这种「零方差组」直接丢弃。

设模型在某难度的答对概率为 p。组内奖励方差是 p(1−p),在 p=0.5 时最大; 「这一组有对有错(可用)」的概率为 U = 1 − pᴳ − (1−p)ᴳ,同样在 0.5 最大。直观代入(G=8):

p=0.95（太易）→ 整组全对 ≈ 0.95⁸ ≈ 66% 的组白采 p=0.50（适中）→ 全对或全错各 ≈ 0.4% → 99% 的组有效 p=0.10（太难）→ 整组全错 ≈ 0.9⁸ ≈ 43% 的组白采

结论:固定难度必然失败。太易 → 很快全对、信号枯竭;太难 → 长期全错、几乎不更新。 只有把成功率维持在 0.5 附近,梯度信号才持续存在。这正是「可验证环境 + 自适应难度」这条路线要解决的核心。

02

原方法:SCALER 怎么做、用什么数据

数据 — 从真实代码题合成可验证环境

SCALER 把真实竞赛编程题(题面 + 参考解,来自 code_contest 一类数据)自动合成为「可验证环境」。每个环境包含:

  • 可验证:带参考解 / 单元测试,运行后得到标准答案;判分由 SandboxFusion 代码沙箱执行(参考解多为 C++)。
  • 可控难度:一个 scale 参数(如规模 n)离散成难度等级 d;difficulty_dict 把 d 映射到具体规模。
  • 无限实例:用随机测例生成器,在任意难度下生成无穷多道题。

已开源环境池:SCALER-{8, 64, 512, 2739}.json(由约 4973 道题合成 2739 个环境);评测集:MATH-500 / AMC23 / AIME24 / MMLU-Pro / BBEH / GPQA。

难度控制器 — 一个固定定点的反馈调节

SCALER 维护一个连续难度 d,用观测到的成功率做比例反馈,把它拉向一个固定目标:

err = success_rate − target(=0.5) d ← clip( d + k · err , [d_min, d_max] )

问题在哪(本 proposal 的出发点)

  • 没有显式目标:它是一个 setpoint 调节器,没有一个被优化的目标函数,因而难以分析、难以推广。
  • 0.5 是写死的:对二值奖励 0.5 恰好最优,但奖励一旦变成「通过测例比例」等连续形式,最优点就会偏离 0.5——定点无法自适应。
  • 塌成单点、缺乏探索:只盯一个难度点,没有显式的「多样性 / 探索」项,也不会主动放弃已饱和的难度。
  • 难度与环境选择割裂:环境之间怎么分配算力,用的是另一套启发式(难度斜率),与难度控制不在同一目标下。
03

如何深入:把难度选择写成自由能最小化

核心思路:不要盯住一个最优点,而是在所有难度档上选一个采样分布 q(d),让它最小化一个借自统计力学的自由能泛函。 先把每个难度的「学习价值」(能量的反面)定义为上面的可用组概率:

U(d) = 1 − p(d)ᴳ − (1−p(d))ᴳ # 学习价值,p=0.5 处最大

把 −U 当作能量,在分布 q 上写出自由能:

F[q] = − Eq[U(d)] − T · H[q] # H 为熵, T 为温度 └─ 能量:概率投向高价值难度 └─ 熵:保持多样性/探索

为什么叫「自由能」,以及它给了什么

自由能 = 能量 − 温度 × 熵,是物理里在「降低能量」与「保持无序(熵)」之间权衡的标准量。 对它求极小(在 Σq=1 约束下用拉格朗日乘子),得到的最优分布正是 Gibbs / 玻尔兹曼分布:

q*(d) ∝ exp( U(d) / T ) F[q*] = −T · log Σd exp(U(d)/T)
三个极限 — 为什么这是「推广」而非「替换」
  • T → 0(低温):概率全压到 argmax U(d) ≈ 成功率 0.5 那一档 → 精确退化回 SCALER 的定点。
  • T → ∞(高温):各难度均匀采样 → 最大探索 / 难度多样性。
  • 中间 T:既聚焦最有信息的难度,又探索邻近难度;并自动放弃 p→1(饱和)与 p→0(学不动)的难度,因其 U 极低。

顺带统一了环境选择。若把价值写成 U(e,d)(环境 e、难度 d),同一推导给出 q*(e,d) ∝ exp(U(e,d)/T), 其环境边缘分布 q*(e) ∝ Σ_d exp(U(e,d)/T) 就是环境采样权重(等于该环境的负自由能)。 于是「环境内难度控制」与「环境间算力分配」由同一个目标、同一个温度旋钮导出。

04

具体怎么操作:在线控制器 + 落地

真实的 p(d) 未知,故在线估计。每个训练步:

1. 由 q*(d) ∝ exp(U(d)/T) 采样本步各难度 # 出题 2. rollout + 沙箱判分,得各难度的成功率 # 观测 3. p̂(d) ← EMA 更新,重算 U(d) # 学习价值 4. 退火温度 T: T₀ → T_min # 逐步聚焦

工程落地(已实现,零基础设施改动)

  • 实现为 SCALER 难度模块的 drop-in 控制器,完全复用其接口(propose_distances / update / 序列化),可直接读取 SCALER 现成的环境 JSON。
  • 用环境变量 DIFFICULTY_MODE=freeenergy 切换:不设则逐字保持 SCALER 原行为(对照公平)。
  • 超参经环境变量注入(G、T₀、T_min、退火步数、EMA),无需重生成数据。
05

实验设计与验证

数据划分

从 SCALER-8 取 5 个训练环境,留 3 个 held-out 环境;另在 6 个公开 benchmark 上评训练前/后,衡量跨分布泛化。

五个对照臂(同模型、同算力,只换难度策略)

臂难度策略检验的假设
static-易难度焊死(低)很快全对 → 信号枯竭
static-中难度焊死(中)固定点的中庸表现
static-难难度焊死(高)长期全错 → 学不动
SCALER-adaptive定点 0.5(基线)维持中等难度的信号
free-energy(本工作)q(d)∝exp(U/T)信号最高 · 泛化最好 · T→0 复现基线

核心指标

  • 有效样本率(机理指标):每步「有对有错」的组占比 = 1 − pᴳ − (1−p)ᴳ 的实测均值。预期 static 很快塌向 0、adaptive 维持、free-energy 最高。
  • 泛化:训练前后在 held-out 环境 + 6 benchmark 上的准确率差。
  • 难度轨迹:static 为平线、SCALER 收敛到 0.5 带、free-energy 为一个随温度收缩的分布。
  • 温度消融:扫 T,并数值验证 T→0 复现 SCALER。
初步验证(合成策略,同日志格式)
有效样本率:static-易 0.22 · static-难 0.49 · 自适应 0.99。 与理论预期一致——自适应把学习信号维持得最高。真实 SCALER 上的 free-energy 臂正在跑,数据回填后更新此处。

诚实的边界 / 失败情形

当模型能力超过 d_max,所有难度都饱和、U→0,任何控制器都无信号可调——这反过来说明需要扩充环境(env scaling),呼应 RLVE/SCALER 的核心论点,也是本工作的自然延伸。

06

贡献小结

  • 指出并形式化 SCALER 定点控制器的局限,把难度选择重述为自由能最小化,给出闭式 Gibbs 解与三个极限(含 T→0 复现基线)。
  • 同一目标统一了环境内难度控制与环境间算力分配。
  • 提供 drop-in、可切换、可复现的实现,以及以有效样本率为机理指标的五臂对照实验设计。
